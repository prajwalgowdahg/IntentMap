Rewrite `src/adapters/react.ts` completely. All three hooks must handle destroyed instances gracefully by checking `im.isAlive` before calling core methods and returning early with `console.warn('[intentmap] ...')` if destroyed.

**useIntentMap** (RCT-01):
- Signature: `useIntentMap(config: IntentConfig, key?: unknown): IntentMapInstance`
- Use `useRef` to store the IntentMapInstance
- Use a second `useRef` to store the config (via `useRef(config)`) so config object identity changes don't trigger re-creation
- Use `useRef` for the key to compare previous vs current: if `key` changed (and previous was defined), destroy old instance and create new one with the current config ref
- `useEffect` cleanup calls `instance.destroy()` -- guard with `isAlive` check since destroy is idempotent but we want to avoid the warning
- Always return the instance (never null)

**useIntent** (RCT-02):
- Signature: `useIntent(im: IntentMapInstance, intent: string, handler: IntentHandler): void`
- BREAKING: Remove the `deps` parameter entirely. The useRef wrapper makes it unnecessary.
- Use `useRef` to store the latest `handler` reference: `const handlerRef = useRef(handler)` then `handlerRef.current = handler` on every render
- Create a stable wrapper function that reads from the ref: `useRef((result: MatchResult, event?: Event) => handlerRef.current(result, event))`
- In `useEffect`, subscribe via `im.on(intent, stableWrapper)` with dependency array `[im, intent]` only -- no handler, no deps
- On cleanup, call the returned unsubscribe function
- Guard: if `im.isAlive` is false at subscription time, log warning and return early (no-op cleanup)

**useIntentBind** (RCT-03):
- Store the unbind function returned by `im.bind(element, ...)` in a ref
- In a `useEffect` cleanup, call the stored unbind function
- The wildcard `im.on('*', ...)` subscription already returns an unsubscribe -- verify it's called in cleanup
- Guard all `im.bind()` and `im.on()` calls with `im.isAlive` checks -- if destroyed, log warning and return early
- Keep the `ref` callback pattern (bindRef) but ensure the unbind from `im.bind()` is captured and cleaned up

After rewriting hooks, update `examples/SearchBar.tsx`:
